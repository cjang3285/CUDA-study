# 신용카드 사기 탐지 — GPU 그래프 방법론 (RAPIDS)

*(영문 원본: [README.md](README.md))*

데이터셋: [Kaggle Credit Card Fraud (ULB)](https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud) — 거래 284,807건, 사기 492건(0.17%), 피처 `V1`~`V28`(PCA 익명화), `Amount`, `Time`.

스택: cuDF / cuML / cuGraph (RAPIDS), WSL2, RTX 4060. 플롯 포함 전체 리포트: `results/report.html`.

## 목표

**GPU 가속 그래프 유사도 기법**으로 사기 거래를 찾아낸다. `Class` 라벨은 그래프를 만드는 데는 쓰지 않고, 결과가 나온 뒤 채점하는 용도로만 쓴다.

## 성공한 것

### 1. kNN 유사도 그래프 + Louvain 커뮤니티 탐지 — 핵심 결과

`knn_louvain.py`: 표준화된 `V1`~`V28`에 대해 15-최근접이웃 그래프를 만들고 cuGraph Louvain 실행.

- Modularity 0.957, 138개 커뮤니티, GPU 연산 총 10초.
- **138개 중 단 2개 커뮤니티(462개 노드)가 사기 비율 80~85%** — 전체 사기의 **77%(492건 중 380건)**를 포착. 그래프 구축 단계에서 라벨은 전혀 안 썼고 순수 피처 유사도만으로 나온 결과.
- 시각적으로도 확인: `umap_check_communities.py`가 cuML UMAP으로 같은 데이터를 2D에 투영하고 이 두 커뮤니티만 색칠해보면, 정상 거래 덩어리에서 뚜렷이 떨어진 반도 모양으로 뭉쳐 있음(`results/report.html` 참고).

**왜 됐나**: 이 사기 그룹은 `V1`~`V28` 공간에서 서로 닮은 "패거리형" 패턴을 공유하고 있어서, 커뮤니티 탐지가 원래 잘하는 일과 정확히 맞아떨어짐.

### 2. 지도학습 XGBoost를 2단계로 — 나머지 회수에 가장 효과적

`fraud_classifier.py`: 남은 사기 ~110건은 서로 뭉치지 않는다(아래 참고). 그래서 `V1`~`V28` + `Amount` + `Time`으로 GPU XGBoost를 학습, 578:1 불균형 대응으로 `scale_pos_weight` 적용.

- PR-AUC 0.83 (이 유명 데이터셋의 문헌 벤치마크와 일치).
- recall 80% / precision 85% 지점: 테스트셋 85,443건 중 140건 플래그, 148건 중 119건 잡음, 오탐 21건.
- Louvain이 놓친 사기의 **29%를 회수** — 아래 두 비지도 시도보다 나은 성적.
- 추가 테스트: 그래프에서 뽑은 kNN거리 피처 추가(PR-AUC 0.8367, 미미한 개선), 상위 5개 중요 피처끼리의 교호작용항 + 시간대 + log(Amount)(PR-AUC 0.8329, **개선 없음** — 그래디언트 부스팅 트리가 이미 분기를 통해 피처 상호작용을 스스로 학습하기 때문에 손으로 만든 교호작용항은 중복 정보).

## 실패한 것 (이유도 기록으로 남김)

### 3. Local Outlier Factor (GPU에서 직접 구현한 LOF)

`investigate_missed.py` (LOF 스크립트는 삭제, git 히스토리 참고) — 같은 k=15 이웃 거리로 로컬 밀도 이탈도를 계산.

- 평균적으로는 실제 신호 있음: 놓친 사기의 LOF 중앙값 1.29 vs 정상 1.09.
- **탐지기로는 못 씀**: LOF로 순위를 매겨서 상위 5,000건(전체의 1.8%)을 플래그해도 놓친 사기의 12%밖에 못 잡음. 정상 거래 중 극단적으로 LOF가 튀는(최대 165) 긴 꼬리가 사기 신호를 덮어버림.
- 그 전에 실제 버그부터 고쳐야 했음: 이 데이터셋에 `V1`~`V28` 기준 완전 동일한 행이 9,144개 있어서 거리 0 → reachability-distance 공식에서 `1/0 = inf` 발생. LOF 계산 전 중복 제거로 해결.

### 4. Personalized PageRank (그래프 확산 / 라벨 전파)

`ppr_label_propagation.py` — 사기 비율이 높은 두 커뮤니티만 시드로 놓고 PageRank 실행(놓친 사기 106건은 시드에서 제외, 순수 홀드아웃 평가). 로컬하게 머물도록 alpha를 0.85→0.05까지 스윕.

- 모든 alpha에서 실패 (최선의 경우도 상위 5,000건 중 3/106건).
- **원인**: kNN 그래프 안에 소수의 "허브 노드"(다른 많은 점들이 자주 "가장 가까운 이웃"으로 뽑는 인기 노드)가 있어서, 시드와의 근접성이나 damping factor와 무관하게 과도한 PageRank 점수를 먹음. 이게 사기 시드에서 나오는 진짜 확산 신호를 덮어버림.

**두 실패에서 얻은 결론**: 남은 ~106~111건의 사기는 단순히 "군집이 잘 안 되는" 정도가 아니라, **서로 다른 두 가지 구조적 이유로 두 개의 독립적인 비지도 신호(밀도, 확산) 모두에서 정상 거래와 구분이 안 됩니다.** 이는 하나의 실패보다 훨씬 강한 증거이며, 이 구간은 더 나은 비지도 그래프 트릭이 아니라 라벨 정보가 필요하다는 뜻입니다.

## recall 100%가 가능한가?

직접 확인함: `V1`~`V28` + `Amount`가 완전히 같은데 라벨이 다른 행은 **0건** — 그러니 이론적으로 못 넘는 벽은 없음. 하지만 학습된 분류기로 테스트셋의 148건 사기를 전부 잡으려면 **테스트 거래 85,443건 중 83,952건(98.3%)**을 플래그해야 함 — 사기 케이스 하나가 전형적인 정상 거래와 점수상 구분이 안 되기 때문. 이건 모델을 고쳐서 될 문제가 아니라, 이 피처셋 자체의 속성.

## 최종 권장 방법론

**2단계 파이프라인**, 각 단계가 원리적으로 잘하는 일을 맡김:

1. **Louvain 커뮤니티 탐지** (비지도, GPU, ~10초) — 순수 피처 유사도만으로 사기의 77%를 80~85% precision으로 포착. 저렴하고, 설명 가능하고, 라벨 불필요.
2. **지도학습 GPU XGBoost**를 그 위에 — 나머지 잔여분을 추가로 회수(PR-AUC 0.83, 1단계가 놓친 것의 29% 회수). 30개 피처 전부를 라벨과의 실제 상관관계로 가중치를 줄 수 있기 때문.

**구조적 한계**: 이 데이터셋엔 카드/사용자 ID가 없어서, 실무 사기탐지에서 가장 강력한 신호(거래 속도, 그 카드의 평소 소비 패턴 대비 편차)를 만들 수가 없음. 이건 데이터 수집 단계의 한계이지, 피처 엔지니어링이나 더 정교한 그래프 알고리즘으로 메울 수 있는 문제가 아님.

|---|---|
| `knn_louvain.py` | 표준화된 V1-28로 kNN 그래프 구축, Louvain 실행, `Class`로 커뮤니티 채점 |

| `investigate_missed.py` | 진단용: 잡힌 사기 vs 놓친 사기 vs 정상의 kNN거리("얼마나 국소적으로 이상한가") 비교 |
| `ppr_label_propagation.py` | Personalized PageRank 시도 (실패 결과, 기록용으로 보존) |
| `fraud_classifier.py` | GPU XGBoost 2단계 분류기, 피처셋 비교, recall/precision 트레이드오프 표 |
| `results/report.html` | 시각 자료(UMAP 플롯, 커뮤니티 표) |
